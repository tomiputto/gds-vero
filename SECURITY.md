# Security Policy

## Reporting a vulnerability
If you discover a security vulnerability in this repository, please report it responsibly.

1. **Do not open a public issue immediately.**
2. **Describe the vulnerability clearly** and include:
   - what you observed,
   - affected packages/apps (if known),
   - steps to reproduce (minimum necessary),
   - logs/screenshots and any relevant stack traces,
   - a proposed fix or mitigation (if you have one).
3. **Include an impact assessment** (for example: data exposure, auth bypass, remote code execution, etc.).

## Where to send reports
Please contact the maintainers via GitHub:
- Open a **private security advisory**: https://github.com/tomiputto/gds-vero/security/advisories/new

## Confidentiality
Reports should remain confidential until:
- a fix is released, or
- the maintainers acknowledge the report and request public disclosure, or
- you receive guidance that disclosure is safe.

## Triage and timelines
Once received, we will:
- acknowledge receipt,
- triage severity and scope,
- request additional info if needed.

We aim to provide an initial response within **48–72 hours**. Fix and disclosure timelines depend on severity and complexity.

## After a fix
After mitigation is available, we will:
- prepare a security advisory with appropriate credit,
- publish details only after the community is protected (when appropriate).

## Dependencies
This project uses automated dependency management and CI checks where possible. For vulnerabilities in dependencies, include:
- the affected package name and version,
- whether the issue is in our code or a third-party dependency.
